## Section 1.1 — Setting the Scene

Every great chapter opens with a hook. Give the reader a reason to keep
turning pages. This paragraph is a placeholder — delete it and write
something that belongs to your story or argument.

leaving your keyboard. You can link to [Pandoc's documentation](https://pandoc.org)
or reference other sections naturally.

## Section 1.3 — Code and Technical Content

If your book includes code examples, fenced blocks work perfectly:

```python
def greet(name: str) -> str:
    """Return a greeting string."""
    return f"Hello, {name}!"
```

Pandoc applies syntax highlighting automatically in HTML and PDF output when
a syntax-highlight theme is available.

## Chapter Summary

End each chapter with a brief summary of what was covered and a transition
toward the next chapter. This helps readers consolidate what they have learned
and builds anticipation for what comes next.
